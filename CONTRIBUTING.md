# Contributing

Thanks for your interest in contributing! All contributions are welcome.

## Bug Fixes

1. Check if an issue already exists, if not create one
2. Fork the repo and create your branch
3. Fix the bug
4. Raise a MR with:
   - Reference to the issue (e.g., "Fixes #12")
   - Brief description of the problem
   - How your change solves it

## New Features

1. Raise an issue first to discuss the feature
2. Fork the repo and create your branch
3. Implement the feature
4. Raise a MR with:
   - Purpose of the feature
   - Related issues (if any)
   - Benefits it provides
   - Any breaking changes

## Guidelines

- Keep code simple and readable
- Test your changes before submitting
- Be polite and humble in discussions
- Respect others' opinions and feedback
- We're all here to learn and help each other
